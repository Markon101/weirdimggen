# Architecture Notes: Latent Distillation to Photos (Rust Rewrite)

## Overview
The goal of this architecture update was to transition from a direct mapping of "chaos" (fractal noise) to colors, towards a more structured "distillation" of a latent space. This allows the generator to "capture" the latent space and intelligently decide how features manifest in the final image.

## Key Changes & Neural Components
1. **Rust Port for Performance:** The entire codebase was rewritten in Rust, utilizing the `rayon` crate for massive parallelization across CPU cores. The previous Python implementation relied on NumPy vectorization, but the custom iterative cell operations required here are much faster when explicitly parallelized in compiled Rust.
2. **KAN (Kolmogorov-Arnold Network) Feature Extraction:** Instead of using static linear weights for feature extraction, the network processes the raw chaotic latent vectors through a cubic B-spline. This allows the network to learn (or in this case, randomly initialize) complex non-linear mappings for the base noise features.
3. **Titans / LSTM Memory Distillation:**
   - The generator iterates over multiple "layers" (analogous to time steps).
   - At each layer, it extracts a feature from the latent space using the KAN spline.
   - It maintains a `Cell State` (memory) and a `Hidden State` (output), similar to an LSTM or the persistent memory mechanisms in the Titans architecture.
   - Forget, Input, and Output gates determine how much of the new chaotic feature is integrated into the structural memory, and how much previous structure is retained.
   - This "distills" the raw chaos into a coherent structural feature vector (`h_state`) rather than just blending noise layers additively.
4. **Domain Warping Feedback Loop:** The previous hidden state directly warps the sampling coordinates of the next layer's latent space, creating a dynamic feedback loop where the network's structural understanding alters its perception of the latent space.
5. **Multi-Frequency Palette Generation:** The distilled hidden state is then passed into a complex trigonometric palette generator, preserving the rich color banding of the original Python script while benefiting from the smoother, memory-integrated structural gradients.

## Future Exploration
- Implement proper learned weights for the KAN splines and LSTM gates using a small autograd engine if specific styles need to be trained.
- Allow for larger hidden state vectors per pixel (e.g., `vec3` or `vec4`) for multi-dimensional feature distillation.

## Updates (Candle ML, SIREN, & Online Learning)
1. **Candle Integration:** Replaced the bespoke cell-based implementation with a proper machine learning backend using Hugging Face's `candle-core` and `candle-nn`. This ensures high performance, native tensor operations, and industry-standard safetensors serialization.
2. **SIREN Coordinate Network:** The architecture was upgraded to a Coordinate-based Neural Network (Continuous Implicit Neural Representation) using Sine activations (SIREN). This is state-of-the-art for generating complex, multi-frequency imagery from spatial coordinates $(x, y)$.
3. **Online Learning Dynamics:** The network now evolves dynamically. Each time it runs, it generates a new random fractal noise target and performs a fast online training phase (20 steps of AdamW optimization). This slowly shifts the model's weights toward the new pattern, creating a constantly evolving latent memory across consecutive executions.
4. **Safetensors Storage:** Weights are natively saved to `model.safetensors`, retaining the evolving aesthetic memory safely and securely between runs.

1. **Snapdragon Thread Optimization:** The engine is now explicitly configured to use **6 threads**, matching the high-performance core configuration of the Snapdragon processor for maximum throughput without thermal throttling.
2. **Weight Persistence:** Neural parameters (KAN spline control points, LSTM gate weights, and palette frequencies) are now serialized to `weights.json` in the project folder. This allows for reproducible art generation; if the file exists, the generator loads the same "aesthetic DNA" for future runs unless `--new-weights` is specified.
3. **CLI Feature Parity:** Re-implemented the `--width`, `--height`, `--out`, and `--count` parameters from the original Python script using the `clap` crate for a professional CLI interface.
4. **SD Card Integration:** The default output path has been restored to `/sdcard/noise.png` to maintain consistency with the previous workflow.
